1. การออกแบบระบบจอง (System Rules)
1.1 จำกัดช่วงเวลาเปิดจอง
	- เปิดให้จองล่วงหน้าไม่เกิน 3 วัน (ตามที่กำหนดไว้)
	- ระบบจะ “disable” วัน/เวลาที่เกินช่วงนี้หรือย้อนหลัง

1.2 ตรวจสอบจำนวนที่ว่าง (availability check)
	- แต่ละ slot (วัน+เวลา) มี 20 คัน
	- ถ้าเต็มแล้วต้องไม่สามารถเลือกได้อีก (query check ก่อน confirm)

1.3 กฎการจองต่อผู้ใช้ (per user/vendor)
	- 1 คัน = 1 ชั่วโมง = 1 slot ต่อวัน
	- ไม่ให้คันเดียวจองมากกว่า 1 ครั้งในวันเดียวกัน
	- จำกัดสูงสุด 10 คัน/วัน/ผู้ใช้

1.4 การยืนยันการจอง (commit mechanism)
	- ต้องมีขั้นตอน “Confirm” หลังเลือกเวลาและจำนวนคัน (คล้ายตะกร้า → checkout)
	- ถ้าไม่กดยืนยันในเวลาที่กำหนด (เช่น 5 นาที) slot จะถูกคืนระบบ

2. วิธีป้องกันการจองแล้วไม่มา (No-show prevention)
2.1 ระบบ Pre-Authorization (มัดจำ)
	- จองแล้วต้อง “ยืนยันด้วยมัดจำ” เช่น 20–50 บาทต่อคัน (หักจริงถ้าไม่มา)
	- ถ้ามาใช้จริง มัดจำคืน/หักเป็นค่าจอด

2.2 ระบบ Credit/Quota
	- ให้สิทธิ์จอง “Quota” ตามจำนวน เช่น vendor 50 สิทธิ์/เดือน
	- ถ้า no-show จะถูกหักสิทธิ์ไป → เดือนนั้นจองได้น้อยลง

2.3 ระบบ Penalty (Blacklist/Freeze)
	- ถ้า no-show ติดต่อกันเกิน X ครั้ง → ระงับสิทธิ์จอง 7 วัน
	- ใช้เป็นแรงจูงใจให้กดยกเลิกถ้าไม่ได้ใช้

2.4 ระบบ Check-in/Check-out
	- ต้อง “เช็คอิน” เมื่อถึงที่จอดจริง (QR code หรือ staff ยืนยัน)
	- ถ้าเลยเวลาเช็คอิน 15 นาทีโดยไม่มา → ระบบปล่อยที่ว่างให้คนอื่นจอง walk-in

3. การแก้ปัญหาความยืดหยุ่น (Flexibility)
3.1 ให้ลูกค้ายกเลิก/แก้ไขได้เอง
	- ถ้าเปลี่ยนแผนต้องสามารถยกเลิกผ่านระบบได้ง่าย
	- แต่ต้องมีกำหนดเวลา เช่น ยกเลิกได้ไม่เกิน 1 ชั่วโมงก่อนเวลาจอง

3.2 ระบบรอคิว (Waiting List)
	- ถ้า slot เต็ม → ลูกค้าสามารถกด “รอคิว”
	- ถ้ามีการยกเลิก ระบบจะส่งแจ้งเตือนทันทีให้คนที่รอคิวมาก่อนกดยืนยันแทน

4. โครงสร้างข้อมูลที่ควรเก็บ
4.1 ตาราง Reservations
	reservation_id
	user_id/vendor_id
	vehicle_id (unique ต่อวัน)
	date, start_time, end_time
	status: pending / confirmed / checked-in / canceled / no-show
	deposit_flag

4.2 ตาราง Vehicles
	vehicle_id
	plate_number
	driver_info (optional ตอนจอง ใส่ทีหลังได้)

4.3 ตาราง Penalties
	user_id
	no_show_count
	suspension_until

Flowchart การจองที่จอดรถ ตั้งแต่การเลือกวัน/เวลา (Slot) ไปจนถึงการยืนยันการจอง เช็คอิน และการปล่อยคืนระบบ โดยมีขั้นตอนหลักดังนี้:
1. ลูกค้าเลือกวัน+เวลา → ระบบ check availability
2. ลูกค้าใส่จำนวนคัน (max 10) + เลือกรถ
3. ระบบจอง slot แบบ “pending”
4. ลูกค้าต้อง confirm (มัดจำ/กดยืนยัน) ภายใน X นาที → ถ้าไม่ confirm = auto-cancel
5. ถึงเวลาจริง → เช็คอิน → ถ้าไม่เช็คอิน = no-show → หักมัดจำ/หัก quota/ลงโทษ

1. ลูกค้าเลือกวัน/เวลา (Slot) 
2. ใส่จำนวนคัน (max 10) + เลือกรถ
2. ระบบตรวจสอบจำนวนที่ว่าง (สูงสุด 20 คัน)
   - ถ้าเต็ม → ต้องเลือกเวลาอื่น
   - ถ้าว่าง → เพิ่มจำนวนคัน + ข้อมูลรถ
3. ระบบเปลี่ยนสถานะเป็น "Pending"
4. ลูกค้ากดยืนยัน + ชำระมัดจำ ภายใน X นาที
   - ถ้าไม่ยืนยันภายในเวลาที่กำหนด → ยกเลิกอัตโนมัติ
   - ถ้ายืนยันแล้ว → สถานะเป็น "Confirmed"
5. เมื่อถึงเวลาจริง:
   - ถ้าไม่มาเช็คอิน → No-Show (หักมัดจำ / เพิ่มโทษ)
   - ถ้ามาเช็คอินสำเร็จ → ใช้งานจริง
6. เมื่อเช็คเอาท์หรือเลยเวลา → ระบบปล่อยที่ว่างคืน ***
